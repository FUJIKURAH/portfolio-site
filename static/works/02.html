冷蔵庫管理サイト(ver 2.0)

- URL fridge-manager-portfolio.netlify.app/

概要

冷蔵庫の食材を管理するWebアプリケーションのVer2として、既存のVer1をベースに React + TypeScript で再実装を行いました。
Ver1で周囲から得たフィードバックをもとにUIを改善し、デザインはFigma Makeを活用してAIと共に再設計しています。

Reactでの実装を通じて、従来のJavaScript中心の設計との違いを理解し、コンポーネント設計や状態管理への理解を深めることを目的としました。

使用技術

React
TypeScript
HTML / CSS
Vite

担当範囲

フロントエンド実装を担当しました。
バックエンドのデータベース設計はVer1で作成したものを継続利用しています。

工夫した点・取り組み

本プロジェクトはReactの学習を目的としていたため、Ver1の設計やコードを「Reactでどう表現するか」という視点を意識して取り組みました。

特に苦労したのは、どの情報をstate（状態）として管理し、どのコンポーネントに責務を持たせるか という設計部分です。
従来のJavaScriptによる実装とは異なり、コンポーネント単位で責務を分割する必要があるため、最初は「どこから実装を始めればよいのか」が分からず試行錯誤しました。

その中で、以下の点を軸に設計を整理しました。
・画面の変化に関わるデータをstateとして定義する
・stateの管理場所を起点にコンポーネント構造を検討する
・不要な再レンダリングや無駄な処理を減らす設計を意識する

AIを活用しながら設計の相談を行い、自分なりに最適な構成を模索しました。
まだ改善の余地はあると感じていますが、Reactにおける状態管理と設計思考を大きく学べたプロジェクトになりました。
